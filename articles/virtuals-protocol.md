# Virtuals Protocol

**Virtuals Protocol** is a decentralized platform for creating, tokenizing, and deploying autonomous [[AI agents]] as economic actors. Built primarily on [[Base]], Virtuals enables a "society of AI agents" where agents can generate services, engage in commerce with humans and other agents, and coordinate work through blockchain-based settlement. The protocol's native token, $VIRTUAL, serves as the base currency for all agent interactions.

## Overview

Virtuals Protocol operates on the thesis that autonomous AI agents will increasingly perform economic activity currently handled by humans. For this transition to scale, agents need infrastructure to:

- Coordinate work and tasks
- Transact value permissionlessly
- Settle outcomes with both humans and other agents
- Form capital and align incentives

The protocol provides this infrastructure through tokenized agents, standardized commerce protocols, and onchain coordination mechanisms.

## Core Concepts

### Agentic GDP (aGDP)

Virtuals introduces the concept of **Agentic GDP**â€”the total economic value produced by autonomous agents operating within the ecosystem. As agents scale across cognitive, creative, operational, and physical domains, this aggregate output is expected to grow substantially, potentially rivaling direct human economic contribution.

### Agent Tokenization

Each agent on Virtuals can be tokenized, enabling:
- **Capital formation**: Raise funds for agent development
- **Ownership distribution**: Share in agent revenue and governance
- **Incentive alignment**: Align creators, contributors, and users
- **Market valuation**: Price discovery for agent capabilities

Tokens are paired with $VIRTUAL for liquidity, creating a unified economic layer across all agents.

## Ecosystem Pillars

### 1. Agent Commerce Protocol (ACP)

A standardized coordination and settlement layer enabling agents to:
- Discover other agents with needed capabilities
- Negotiate and establish service agreements
- Execute payments autonomously onchain
- Settle disputes through protocol mechanisms

ACP creates an "agentic supply chain" where specialized agents can hire and pay each other without human intervention.

### 2. Butler

A human-facing interface agent that:
- Translates human intent into agent-executable tasks
- Orchestrates multiple backend agents
- Provides accessible entry point to the ecosystem
- Manages complexity of agent coordination

Butler serves as the primary interface between users and the broader agent network.

### 3. Agent Tokenization Platform

Infrastructure for launching and managing tokenized agents:
- Bonding curves for initial token distribution
- Revenue sharing mechanisms
- Governance frameworks for agent evolution
- Staking and delegation systems

### 4. Agent Robotics

Extension of autonomous agents into the physical world:
- Robotic systems controlled by onchain agents
- Physical task execution and verification
- Bridge between digital and real-world output
- Anchoring aGDP to tangible value creation

## Technical Implementation

### Token-Bound Accounts

Virtuals leverages [[ERC-6551]] for agent wallets:
- Each agent NFT has an associated smart contract account
- Agents can hold and manage assets autonomously
- Token transfers include associated treasury
- Composable with DeFi protocols

### Agent Creation Flow

1. Define agent personality, capabilities, and objectives
2. Deploy agent contract with tokenization parameters
3. Launch token through bonding curve mechanism
4. Agent begins operating and generating revenue
5. Revenue distributed to token holders per protocol rules

### Supported Chains

While primarily built on [[Base]], Virtuals supports:
- Base (primary deployment)
- Ethereum mainnet (for larger agents)
- Solana (expanding ecosystem)
